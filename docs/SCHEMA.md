# BuildWithLami — Database Schema Reference

> **Generated:** 2026-07-11 (Phase 12 — Schema audit & cleanup)
> **Engine:** PostgreSQL 14+ (Supabase / Vercel Postgres)
> **Source of truth:** `backend/src/sql/` + `backend/migrations/v2_*.sql` … `v20_*.sql`
> **Status:** ✅ All migrations idempotent, all code references match the schema.

This document is the canonical reference for every table, column, index, and constraint
in the BuildWithLami production database. It is generated by reading the SQL files in
order and the application code that reads/writes each table.

If you change the schema, update this document in the same PR.

---

## How to read this document

- **PK** = Primary key (UUID, default `gen_random_uuid()` unless noted)
- **FK** = Foreign key (cascades noted explicitly)
- **Idx** = Index (named `idx_<table>_<columns>` unless noted)
- `?` after a column name means it is nullable
- All `TIMESTAMPTZ` columns default to `NOW()` unless noted
- All `UUID` primary keys use `gen_random_uuid()` (requires `pgcrypto`)

## 14. Applying the schema

```bash
# 1. Fresh database — runs init.sql + createMissingTables.sql
npm --prefix backend run db:init

# 2. Apply all v2..v20 migrations (idempotent)
npm --prefix backend run db:migrate

# 3. Seed the first admin user
npm --prefix backend run db:seed

# 4. (Optional) seed the two default intake templates
npm --prefix backend run db:templates
```

Every migration is idempotent — re-running `db:migrate` on an already-migrated
database is a no-op.

### Verifying after a migration

```sql
-- Should return 22 tables (no `conversations`).
SELECT count(*) FROM information_schema.tables
WHERE table_schema = 'public' AND table_type = 'BASE TABLE';

-- Should return 56 indexes.
SELECT count(*) FROM pg_indexes WHERE schemaname = 'public';

-- The v20 migration should be the last applied.
SELECT * FROM (VALUES ('v2'), ('v3'), ('v4'), ('v5'), ('v6'), ('v7'),
                     ('v8'), ('v9'), ('v10'), ('v11'), ('v12'), ('v13'),
                     ('v14'), ('v15'), ('v16'), ('v17'), ('v18'), ('v19'),
                     ('v20')) AS m(version)
WHERE NOT EXISTS (
    SELECT 1 FROM information_schema.columns
    WHERE table_name = 'invoices' AND column_name = 'invoice_number'
);
```

If any of these checks fail, re-run `npm run db:migrate` — every migration
uses `IF NOT EXISTS` and `DO $$` guards, so re-application is safe.
